3. SCRAPING METHODOLOGY
================================================================================

3.1 Target Website

Platform: Banggood.com
Categories Scraped: Electronics, Home & Garden, Fashion, Sports & Outdoors,
                    Toys & Hobbies

3.2 Data Points Collected

The scraping process extracted the following information for each product:

- Product Name: Full product title
- Price: Product price in USD
- Rating: Customer rating (1-5 scale)
- Reviews: Number of customer reviews
- Link: Direct URL to product page

3.3 Technical Approach

Tools Used: Python with BeautifulSoup, Requests, and Selenium libraries

Process Overview:
1. Identified target product pages and HTML structure
2. Implemented automated navigation across multiple category pages
3. Extracted product data using CSS selectors and HTML parsing
4. Handled dynamic content loading with Selenium where necessary
5. Implemented respectful scraping practices with rate limiting

3.4 Challenges & Solutions

Challenge                         | Solution Implemented
----------------------------------|------------------------------------------
Dynamic JavaScript content        | Used Selenium for rendering JavaScript-
                                  | heavy pages
Rate limiting by website          | Added 2-3 second delays between requests
Pagination handling               | Automated page navigation through URL
                                  | parameters
Inconsistent data formats         | Created standardization functions for
                                  | parsing
Connection timeouts               | Implemented retry logic with exponential
                                  | backoff

3.5 Ethical Considerations

- Respected robots.txt file guidelines
- Implemented reasonable delays between requests (2-3 seconds)
- Used appropriate user-agent headers
- Scraped only publicly available information
- No authentication bypass attempts

4. DATA CLEANING & TRANSFORMATION
================================================================================

4.1 Initial Data Quality Assessment

The raw dataset revealed several quality issues requiring attention:

Issues Identified:
- Missing values in rating and reviews columns
- Duplicate product entries
- Inconsistent price formatting with currency symbols
- Review counts stored as text with commas
- Invalid rating values exceeding 5.0 scale
- Broken or malformed product links

4.2 Cleaning Process

Step 1: Handle Missing Values
- Identified records with missing ratings or reviews
- Filled missing ratings with category-wise median values
- Set missing review counts to 0
- Removed records with missing critical fields (name, price)

Step 2: Remove Duplicates
- Identified duplicates based on product name and link
- Retained first occurrence of each unique product
- Removed 142 duplicate records

Step 3: Standardize Data Types
- Removed currency symbols from price column
- Converted price to numeric float values
- Extracted numeric values from review count text
- Validated rating values to ensure 1-5 scale
- Verified URL formatting for product links

Step 4: Data Validation
- Ensured all prices are positive values
- Confirmed ratings fall within 0-5 range
- Verified review counts are non-negative integers
- Validated all links are properly formatted URLs

4.3 Data Quality Improvement

Metric              | Before Cleaning | After Cleaning
--------------------|-----------------|----------------
Total Records       | 5,142           | 4,987
Missing Values      | 437             | 0
Duplicate Records   | 142             | 0
Invalid Entries     | 13              | 0
Data Completeness   | 91.5%           | 100%

4.4 Final Dataset Structure

The cleaned dataset contains the following columns:
- product_name: Product title/name
- price: Price in USD (numeric)
- rating: Customer rating (0-5 scale)
- reviews: Number of reviews (integer)

6. SQL AGGREGATED INSIGHTS
================================================================================

6.1 Database Implementation

Platform: Microsoft SQL Server
Database Name: BANGGOOD
Table Name: BanggoodProducts

6.2 SQL Analysis Queries

Query 1: Average Price per Price Tier

Purpose: Calculate average price in PKR for each price tier

Query:
SELECT price_tier, AVG(price_pkr) AS avg_price_pkr
FROM BanggoodProducts
GROUP BY price_tier
ORDER BY avg_price_pkr DESC;

Insight: Shows which price tiers offer the most expensive products on average.

Query 2: Average Rating per Price Tier

Purpose: Calculate average customer rating for each price tier

Query:
SELECT price_tier, AVG(rating) AS avg_rating
FROM BanggoodProducts
GROUP BY price_tier
ORDER BY avg_rating DESC;

Insight: Identifies which price tiers maintain highest customer satisfaction.

Query 3: Product Count per Price Tier

Purpose: Count total products in each price tier

Query:
SELECT price_tier, COUNT(*) AS product_count
FROM BanggoodProducts
GROUP BY price_tier
ORDER BY product_count DESC;

Insight: Reveals product distribution across different price tiers.

Query 4: Top 5 Reviewed Items per Price Tier

Purpose: Identify most-reviewed products in each price tier

Query:
SELECT *
FROM (
    SELECT 
        price_tier,
        product_name,
        reviews,
        price_pkr,
        RANK() OVER (PARTITION BY price_tier ORDER BY reviews DESC) AS 
        review_rank
    FROM BanggoodProducts
) ranked
WHERE review_rank <= 5
ORDER BY price_tier, review_rank;

Insight: Highlights most popular products within each price segment based on
customer engagement.

Query 5: Stock Availability Percentage

Purpose: Calculate stock availability using reviews as proxy indicator

Query:
SELECT
    price_tier,
    100.0 * SUM(CASE WHEN reviews > 0 THEN 1 ELSE 0 END)/COUNT(*) AS 
    stock_available_percent
FROM BanggoodProducts
GROUP BY price_tier
ORDER BY stock_available_percent DESC;

Insight: Products with reviews indicate active stock availability, showing
which price tiers maintain better inventory.
